# CVM++ (Custom Virtual Machine & Compiler)

A lightweight, custom scripting language built entirely from scratch in C++. This project demystifies how high-level code translates into machine instructions by implementing a custom Lexer, Parser, Bytecode Compiler, and a Stack-Based Virtual Machine.

## Project Deliverables Achieved
- [x] **Lexer & Parser:** Implements recursive descent parsing to generate an Abstract Syntax Tree (AST).
- [x] **Bytecode Compiler:** Flattens the AST into a raw array of `uint8_t` Operation Codes (Opcodes).
- [x] **Virtual Machine:** Executes the bytecode using a stack-based runtime architecture.
- [x] **File Runner CLI:** Reads `.cvm` scripts and executes them.
- [x] **Matrix/Debug Mode:** Disassembles and prints the generated Bytecode before execution.

## Repository Structure
* `examples/` - Sample `.cvm` scripts demonstrating language features
* `include/` - C++ Header files (AST nodes, Opcodes, class definitions)
* `src/` - C++ Source implementations
* `CMakeLists.txt` - Build configuration

## Language Features
CVM++ is designed to be simple but Turing-complete. It supports:
* **Basic Data Types:** Integers
* **Math Operations:** `+`, `-`, `*`, `/`
* **Comparisons:** `==`, `<`
* **Variables:** Declaration (`let x = 10`) and reassignment (`x = x + 1`)
* **Control Flow:** `if/else` statements and `while` loops
* **I/O:** `print` keyword

## How to Build and Run
This project uses **CMake** and requires a C++14 compatible compiler (like GCC via MSYS2 on Windows).

**1. Build the VM:**
```bash
mkdir build
cd build
cmake .. -G "MinGW Makefiles"
cmake --build .

**2. Compile to Bytecode:**
```bash
.\cvm.exe compile ../examples/04_factorial.cvm factorial.bin

**3. Execute on the VM:**
```bash
.\cvm.exe run factorial.bin
